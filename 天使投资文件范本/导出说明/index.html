Markdown 转 DOCX / PDF 导出说明

本说明用于把 docs/天使投资文件范本/ 下的 Markdown 合同底稿转换为 Word/WPS 可编辑文件或 PDF。正式签署建议先导出 DOCX，经律师修订后再导出 PDF。

1. 推荐路径

目标 推荐方式 命令/操作
DOCX Pandoc pandoc input.md -o output.docx
PDF 先 DOCX 后 Word/WPS 导出 用 Word/WPS 打开 DOCX，检查表格和签署页后导出 PDF
直接 PDF Pandoc + xelatex pandoc input.md -o output.pdf --pdf-engine=xelatex
网站预览 MkDocs mkdocs serve --config-file startup-investement/mkdocs.yml

2. 单个文件转 DOCX

在仓库根目录运行：

pandoc "startup-investement/docs/天使投资文件范本/温和条款/02-投资条款清单-Term-Sheet.md" -o "温和版-Term-Sheet.docx"

3. 单个文件转 PDF

如果本机有中文 LaTeX 环境：

pandoc "startup-investement/docs/天使投资文件范本/温和条款/02-投资条款清单-Term-Sheet.md" -o "温和版-Term-Sheet.pdf" --pdf-engine=xelatex

如果没有中文 LaTeX 环境，先转 DOCX，再用 Word/WPS 导出 PDF。这个路径更稳定，也方便律师修订。

4. Word/WPS 排版建议

元素 建议
一级标题 使用“标题 1”样式
二级标题 使用“标题 2”样式
编号条款 使用 Word 自动多级编号，避免手工编号错乱
表格 保留 Markdown 表格，导出后检查列宽
占位符 搜索 [，逐项替换 [公司名称] 等占位符
签署页 每个签署页前插入分页符
说明段落 正式签署前删除“起草说明”“律师审阅提示”等说明性内容

5. 批量导出示例

mkdir -p export-docx
for f in startup-investement/docs/天使投资文件范本/温和条款/*.md; do
  name=$(basename "$f" .md)
  pandoc "$f" -o "export-docx/$name.docx"
done

6. 签署版处理

  1. 删除说明性段落和参考来源。
  2. 补齐交易信息表、附件和披露清单。
  3. 请律师核对公司法、证券监管、税务和争议解决条款。
  4. 最终版转 PDF，并保留可编辑 DOCX 作为谈判记录。
